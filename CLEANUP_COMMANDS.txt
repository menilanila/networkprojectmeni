# CLEANUP COMMANDS - Docker & Virtual Environment

## STOP DOCKER CONTAINERS

### Option 1: Stop but keep containers (can restart later)
```powershell
docker-compose -f "E:\networking project\python_networking\docker-compose.yml" stop
```

### Option 2: Stop AND remove containers (clean slate)
```powershell
docker-compose -f "E:\networking project\python_networking\docker-compose.yml" down
```

### Option 3: Remove containers + images (nuclear option)
```powershell
docker-compose -f "E:\networking project\python_networking\docker-compose.yml" down --rmi all
```

---

## DEACTIVATE PYTHON VIRTUAL ENVIRONMENT

### From any directory:
```powershell
deactivate
```

### If that doesn't work, use full path:
```powershell
& "E:\networking project\python_networking\venv\Scripts\Deactivate.ps1"
```

---

## COMPLETE CLEANUP (Everything at Once)

### Run this in PowerShell:
```powershell
# Navigate to project
cd "E:\networking project\python_networking"

# Stop Docker
docker-compose down

# Deactivate venv
deactivate

# Verify cleanup
Write-Host "Docker containers:" -ForegroundColor Green
docker ps

Write-Host "`nVirtual environment:" -ForegroundColor Green
if ($env:VIRTUAL_ENV) {
    Write-Host "Still active: $env:VIRTUAL_ENV"
} else {
    Write-Host "Deactivated ✓"
}
```

---

## VERIFICATION COMMANDS

### Check Docker containers are stopped:
```powershell
docker ps
```
**Expected:** Empty list (no containers running)

### Check virtual environment is deactivated:
```powershell
$env:VIRTUAL_ENV
```
**Expected:** Blank (nothing shown)

Or check your terminal prompt:
- **Active venv:** `(venv) PS E:\...`
- **Deactivated:** `PS E:\...` (no (venv) prefix)

## QUICK REFERENCE

| Task | Command |
|------|---------|
| Stop containers | `docker-compose down` |
| Remove containers | `docker-compose down` |
| Remove images too | `docker-compose down --rmi all` |
| Deactivate venv | `deactivate` |
| Check Docker status | `docker ps` |
| Check venv status | `$env:VIRTUAL_ENV` |
